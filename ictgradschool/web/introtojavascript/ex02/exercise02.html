Learning about the DOM Part 2

(Lab 7 Exercise 2)

The DOM models an HTML page as a tree structure.

Every component of your HTML page is modelled as a node in the DOM tree.

Warning: attributes are no longer going to inherit methods from Node in the DOM4 specification. Refer to MDN The Attr Object. But that's in the future. At present attributes are still Nodes, even if calling Node methods on an attribute makes little sense.

For each Node in the DOM tree, particularly nodes that are elements, you will want want to think about what node is its parent, whether it has any siblings, and any children.

Knowing these relationships can let you navigate from one Node to another in an HTML page, starting at the document node.

You'll be trying that in the next exercise. For this exercise, you will need to understand the interrelationships between the pieces of HTML content on this page: elements, text nodes, and anything else starting at the document level.
